# Dashboard Runtime Data Contract

The dashboard app consumes compact JSON files generated by
`SGP-Data-Pipeline/01_Code/src/sgp_pipeline/processors/dashboard`.

## Runtime Files

```text
dashboard-app/app/public/data/projects.runtime.json
dashboard-app/app/public/data/cofinancing.runtime.json
dashboard-app/app/public/data/content-profiles.json
dashboard-app/app/public/data/country-aliases.json
dashboard-app/app/public/data/data-dictionary.json
dashboard-app/app/public/geo/world-countries.geojson
dashboard-app/app/public/geo/authoritative-provenance.json
```

`projects.runtime.json` and `cofinancing.runtime.json` use a compact table
shape:

```json
{
  "fields": ["fieldName"],
  "rows": [["field value"]]
}
```

The dashboard source expands those tables into typed records at runtime.

## Refresh Source

Do not edit runtime JSON by hand. Refresh it from:

```text
SGP-Data-Pipeline/02_Inputs/dashboard/raw/
SGP-Data-Pipeline/02_Inputs/dashboard/geo/
```

Run the pipeline commands documented in
`SGP-Data-Pipeline/04_Documentation/dashboard/RUNBOOK.md`.

## Commit Guidance

Commit runtime JSON only when it is intentionally part of the deployed static
frontend state. Large normalized files such as `projects.normalized.json`,
`cofinancing.normalized.json`, `aggregates.json`, and `search-index.json`
belong in `SGP-Data-Pipeline/03_Outputs/`, not the frontend repository.

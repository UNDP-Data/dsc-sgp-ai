# Archive Browser Refresh

The `/archive-browser/` route is a static browser payload generated by the site
archive pipeline. It is committed in the frontend repo because GitHub Pages
serves it directly.

## Source of Truth

The scraper and browser generation code live in:

```text
SGP-Data-Pipeline/01_Code/src/sgp_pipeline/scrapers/site_archive/
```

Prepared static output is published from:

```text
SGP-Data-Pipeline/00_API/archive-browser/static/
```

into:

```text
SGP-KLP-Frontend/archive-browser/
```

## Publish Command

From `SGP-Data-Pipeline/01_Code`:

```bash
PYTHONPATH=src python -m sgp_pipeline api publish
```

This copies the prepared static archive browser payload into the frontend route.

## Commit Guidance

The committed archive browser should contain static route assets and sharded
payload files required by the browser. Do not commit raw scraper downloads or
intermediate scraper outputs to `SGP-KLP-Frontend/`.
